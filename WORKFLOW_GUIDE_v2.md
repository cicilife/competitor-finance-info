## 数据库字段说明

| 列名 | 说明 |
|------|------|
| G | 数据值（原始引用值） |
| H | 单位 |
| N | 归一化指标数值 |
| O | 归一化计算逻辑/折算说明 |

**关键原则**: G列存储原始引用值（如259045百万日元），N列归一化为千元人民币

## 单位换算规则

### 核心公式
```
归一化数值 = 原始值 × 原始单位系数 × 汇率
```

### 示例
- **美元(百万)**: 6586百万美元 × 1,000 × 7.2 = 47,419,200千元
- **欧元(百万)**: 7296.2百万欧元 × 1,000 × 7.8 = 56,910,360千元
- **日元(百万)**: 259045百万日元 × 1,000 × 0.048 = 12,434,160千元
- **瑞郎(百万)**: 511.1百万瑞郎 × 1,000 × 8.1 = 4,139,910千元

## 衍生指标计算

### 运营费用率分解公式
```
运营费用率 = 运营费用 / 营业收入

运营费用率 = 广告及市场费用率 + 员工费用率 + 研发费用率
```

**重要**: 运营费用率可以从两个维度验证：
1. **直接计算**: 运营费用 / 营业收入
2. **拆分加总**: 广告及市场费用率 + 员工费用率 + 研发费用率

两者应该相等或非常接近（差异在0.5pp内），如差异过大需要复核。

## 财年格式标准化

**重要原则**: 归一化周期字段使用自然年(YYYY)，不使用财年结束日期。

| 财年原文 | 报告周期(F列) | 归一化周期(L列) |
|---------|-------------|----------------|
| FY20241231 (12/31结束) | FY20241231 | **2024** |
| FY20250331 (3/31结束，2024年4月-2025年3月) | FY20250331 | **2024** |
| FY20240331 (3/31结束，2023年4月-2024年3月) | FY20240331 | **2023** |
| FY20250531 (5/31结束) | FY20250531 | **2025** |
| FY20260329 (3/29结束) | FY20260329 | **2025** |

**命名规范**:
- 报告周期(F列、L列): 用财年原文`FY+YYYYMMDD`或`FY+YYYY`格式
- 归一化周期(L列): 统一使用**自然年YYYY格式**（即数据所属的自然年）
- 财年跨年时(如3月31日结束)，归一化为**财年开始年**

**示例**:
- Nike FY2024 (2023.6-2024.5) → 归一化为 2024
- 安踏 FY2024 (2023.4-2024.3) → 归一化为 2024
- 安踏 FY2023 (2022.4-2023.3) → 归一化为 2023
- 加拿大鹅 FY2026 (2025.4-2026.3) → 归一化为 2025

## 数据质量检查清单

- [ ] G列原始值与H列单位匹配
- [ ] N列归一化数值 = G列 × 换算系数
- [ ] 报告周期格式正确（FY+YYYYMMDD）
- [ ] 归一化周期使用自然年YYYY格式
- [ ] 区域名称统一（大中华区/中国内地/亚太区/TOTAL）
- [ ] 无重复数据行
- [ ] 原文摘录与数据值匹配
- [ ] 百分比指标（毛利率等）归一化后数值不变
- [ ] 运营费用率 ≈ 广告及市场费用率 + 员工费用率 + 研发费用率

## 常见错误避免

### 错误1: 双重单位换算
**错误**: 百万单位再乘1,000,000
**正确**: 百万单位 × 1,000 = 千元单位

### 错误2: 单位标注与实际不符
**错误**: G列存259045000000，单位标"百万日元"
**正确**: G列存259045，单位标"百万日元"

### 错误3: 财年周期不一致
**错误**: 使用FY2024而非FY20250331
**正确**: 统一使用"FY"+"YYYYMMDD"格式
