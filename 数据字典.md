# 数据字典

## 1. 品牌数据 (brands)

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 1 |
| name | TEXT | 品牌中文名称 | 迪卡侬 |
| english_name | TEXT | 品牌英文名称 | Decathlon |
| category | TEXT | 品牌分类 | competitor |
| focus | TEXT | 关注重点 | 全品类运动零售 |
| created_at | TIMESTAMP | 创建时间 | 2026-05-17 10:00:00 |

**category取值**:
- `core`: 核心分析对象（迪卡侬）
- `competitor`: 直接竞品（安踏、李宁等）
- `channel`: 渠道商（滔搏、宝胜）
- `reference`: 参考品牌（耐克、阿迪达斯等）

## 2. 新闻资讯 (news_items)

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 1001 |
| title | TEXT | 新闻标题 | 安踏发布2025年财报 |
| url | TEXT | 原文链接（唯一） | https://... |
| source | TEXT | 来源媒体 | 新浪财经 |
| brand_name | TEXT | 关联品牌（外键） | 安踏体育 |
| content | TEXT | 新闻正文 | ... |
| content_type | TEXT | 内容分类 | 企业经营 |
| publish_date | TEXT | 发布日期 | 2026-05-15 |
| crawl_date | TIMESTAMP | 抓取时间 | 2026-05-15 14:30:00 |
| summary | TEXT | 摘要（200字以内） | ... |
| tags | TEXT | 标签（JSON数组） | ["企业经营", "财务数据"] |

**content_type取值**:
- `企业经营`: 财务报表、业绩公告、投资者关系
- `新品发布`: 新产品上市、产品线扩展
- `市场推广`: 营销活动、品牌合作、广告投放
- `渠道策略`: 门店拓展、电商布局、合作伙伴
- `其他`: 未分类内容

## 3. PDF报告 (pdf_reports)

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 501 |
| brand_name | TEXT | 品牌名称（外键） | 安踏体育 |
| filename | TEXT | 文件名称 | 2025 Annual Report.pdf |
| file_path | TEXT | 文件路径（唯一） | data/raw/pdf/安踏体育/2025 Annual Report.pdf |
| report_type | TEXT | 报告类型 | 年报 |
| year | TEXT | 年份 | 2025 |
| quarter | TEXT | 季度 | Q4 |
| download_date | TIMESTAMP | 下载时间 | 2026-05-17 10:00:00 |

**report_type取值**:
- `年报`: Annual Report
- `季报`: Quarterly Report
- `新闻稿`: Press Release
- `其他财务资料`: 其他财务相关文档

| 字段 | 类型 | 说明 | 示例 |

## 5. 周度报告 (weekly_reports)

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 1 |

## 6. 数据流向图

```
数据源 → 爬虫模块 → 原始数据(raw) → 处理模块 → 数据库(reports.db) → 报告生成 → 输出报告(output)
    ↓              ↓                      ↓
  RSS/网页      PDF文件               结构化数据
```

## 7. 数据质量规则

### 新闻资讯
- URL必须唯一（避免重复）
- title不能为空
- content长度限制在5000字符以内
- publish_date格式：YYYY-MM-DD 或 YYYY年MM月DD日

### PDF报告
- file_path必须唯一
- brand_name必须在brands表中存在
- report_type必须是预定义值之一

### 时间标准
- 使用北京时间（UTC+8）
- 日期格式统一为YYYY-MM-DD
- 时间格式统一为YYYY-MM-DD HH:MM:SS
